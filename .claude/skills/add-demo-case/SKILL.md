# Sophiate デモ一覧にケースを追加する

`/demos` ページ（[src/DemoShowcase.jsx](../../../src/DemoShowcase.jsx)）にデモ事例を追加するための作業手順。

## 入力として受け取るもの

ユーザーからは通常、以下の組み合わせで渡される：
- デモの公開 URL（必須）
- 案件概要 / 背景 / 要件一覧 / 機能一覧（あれば）
- 「目的や課題はよしなに」と任せられるケースあり

情報が足りなくても、URL と業界がわかればカード1枚は書ける。

## 作業ステップ

### 1. 業界（industry）を決める

既存の業界キーから選ぶ：

| key | 日本語名 | 想定対象 |
|---|---|---|
| `construction` | 建設・インフラ | 現場・資金繰り・BtoB 基幹 |
| `nursing` | 介護・ヘルスケア | 福祉・医療・シフト管理 |
| `education` | 教育・スクール | LMS・研修・eラーニング |
| `hr` | 人材・HR | 採用・配置・適性診断 |
| `sales` | 営業・マーケティング | 営業支援・CRM・LP |
| `lifestyle` | 住宅・生活サービス | 訪問販売・リフォーム・BtoC アプリ |

**既存に収まらなければ新業界を追加**する（手順は §4）。迷ったら主業種で決める。副業要素はタグで補う。

### 2. DEMOS 配列にオブジェクトを追加

`src/DemoShowcase.jsx` の `DEMOS` 配列に、業界ごとの既存カードの並びを壊さないよう **同じ industry のカードの隣** に追加する（`INDUSTRY_ORDER` に沿って表示されるので配列順序は必須ではないが、読みやすさのため合わせる）。

テンプレ：

```js
{
  id: "kebab-case-unique-id",              // URLのスラグ等から決める。重複禁止
  industry: "construction",                 // §1 の key
  title: "短い製品名",                        // 全角 8〜16 文字目安
  subtitle: "1行キャッチ",                    // 記号区切り OK（例: "A × B × C"）
  url: "https://...",                       // 主デモURL（必須）
  secondaryUrl: "https://...",              // 任意。2つ目のURLがある場合のみ
  secondaryLabel: "ユーザー側を見る",         // secondaryUrl を書いたら必須
  tags: ["タグ1", "タグ2", "タグ3"],          // 3〜4個。# は自動付与
  overview:
    "3〜5文で、このデモが何をするか・誰に向けたものか・主要機能を具体的に書く。既存カードの文体（常体・敬体なしの情報密度高め）を真似る。",
  challenge:
    "業界課題を2〜4文で。『なぜこれが必要だったか』を書く。『〜できていない』『〜に依存している』のように現状の痛みを名指しする。overview と重複させない。",
},
```

#### 文体ルール（既存カードに揃える）
- 敬語は使わない。体言止めと常体の混在 OK
- 「〜する」「〜できる」基調。「〜します」「〜です」は避ける
- 業界特有の具体語を入れる（例: 「18名×31日」「IT導入補助金」「全国11拠点・70名体制」）
- `overview` は機能の列挙ではなく、価値とストーリーを書く
- `challenge` は導入文（困りごと） → 現状の代替手段 → その痛み、の3段構成が読みやすい

### 3. id / url の衝突チェック

- `id` は重複禁止（React key に使われる）
- 同じ URL のデモが既にないか grep して確認

### 4. 新業界を追加する場合のみ

`INDUSTRIES` オブジェクトに以下を追記：

```js
newkey: {
  key: "newkey",
  name: "日本語業界名",
  nameEn: "ENGLISH NAME",
  color: "#hexcolor",         // ブランドカラー。既存と色相がかぶらないこと
  colorSoft: "#hexpastel",    // 背景用のごく薄いペールトーン
  colorMid: "#hexmid",        // colorより明るめ。グラデの片側に使う
  icon: (
    <svg viewBox="0 0 48 48" fill="none" width="28" height="28">
      {/* 2〜4パスのシンプルな線画。stroke="currentColor" strokeWidth="2.5" */}
    </svg>
  ),
  tagline: "短い一言（15〜25文字）",
},
```

既存で使われている色（重複回避の目安）:
- construction `#f97316` オレンジ
- nursing `#10b981` 緑
- education `#7b2ff7` 紫
- hr `#0ea5b7` ティール
- sales `#4f6bf5` 青
- lifestyle `#ec4899` ピンク

そして `INDUSTRY_ORDER` 配列の **末尾に追加**：

```js
const INDUSTRY_ORDER = ["construction", "nursing", "education", "hr", "sales", "lifestyle", "newkey"];
```

この配列は画面上でのセクション表示順と、上部の業界チップの並びを制御する。

### 5. 動作確認

```bash
npx vite build
```

ビルドが通ればOK。ブラウザで確認する場合は `npx vite` で起動。

具体的にチェックすべき項目：
- 追加したカードが該当業界セクションに表示されるか
- 業界チップのカウント（`N`件）が正しく増えているか
- CASE / 番号（`CASE / 01` 等）は DEMOS 配列のインデックス+1で自動採番されるので手動調整不要
- 新業界追加の場合、色が既存と衝突していないか（チップのホバー色、カードのリボングラデ）

## やらないこと

- スクショ画像の追加はこのデモ一覧では不要（カードは文字情報のみで構成）
- 新規ファイル作成は原則不要（全て `DemoShowcase.jsx` で完結）
- CSS 側の追加も不要（CSS変数 `--ind-color` 等で新業界は自動対応する）

## ユーザーから「目的や課題はよしなに」と任されたとき

URL だけ渡され中身の案件概要がない場合は、以下の順で情報を補う：
1. まず渡された URL を `WebFetch` で開き、ページタイトル・主要見出し・CTAから対象領域を推定
2. 推定した業界と中心機能を **ユーザーに口頭で確認しないまま** 書き進めてよい（Auto mode なら特に）
3. 既存カードの `challenge` 文体を見て、同種業界で書かれている痛み構造をベースに具体化する

書いた内容に不安があれば、追加後に「こう解釈しました。違えば差し替えます」と1行添えて投げる。
